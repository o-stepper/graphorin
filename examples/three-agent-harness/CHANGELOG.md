# three-agent-harness

## 0.1.0

Initial release of the three-agent orchestration harness example. Wires three `createAgent({...})` calls (Planner / Generator / Evaluator) through `evaluatorOptimizer({...})` with the canonical structured-handoff convention (`<artifactRoot>/<runId>/progress/<role>.<seq>.txt`) backed by `createProgressIO({...})`, ships a deterministic per-role stub provider so the smoke test runs hermetically in well under 30 s, exposes an opt-in `runResearchAndDecideVariant({...})` that uses `Agent.fanOut({...})` with a `'judge-merge'` strategy and three stubbed web-search siblings under `maxConcurrentChildren: 4`, and ships a minimal in-tree `createCitationAgent({ sources, mode: 'inline' })` helper that binds claims to sources via substring overlap and emits `agent.citation.bound.total` / `agent.citation.unbound.total` counters when a `Tracer` is wired. The `recipe: 'stub'` default keeps CI offline; `recipe: 'ollama'` swaps in `ollamaAdapter({...})` for a real local-LLM run. Documents the workflow-integration stretch goal (`Dispatch(...)` outer + `Agent.fanOut(...)` inner) and the recommended `.gitignore` entry (`.graphorin/progress/`) for project-tier artifacts.
