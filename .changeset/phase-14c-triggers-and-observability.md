Phase 14c — triggers daemon, consolidator daemon, extended health rollup, Prometheus metrics, scope-enforced replay endpoints, and audit-chain verification land on top of the Phase 14a + 14b foundations. Phase 14 (`@graphorin/server`) is now feature-complete; the runtime spec's "first-class daemon" promise (DEC-011) is fulfilled end-to-end.

`@graphorin/server` extensions:

- **Triggers daemon (`@graphorin/server/triggers`).** New `createTriggersDaemon({ scheduler })` adapter wires the `@graphorin/triggers` `Scheduler` into the server lifecycle: `beforeStart` calls `scheduler.start()` (which loads every persisted row from `trigger_state` and schedules the next fire honouring per-trigger `catchupPolicy`); `beforeShutdown` calls `scheduler.stop()` so cron / interval / idle / event triggers survive process restarts as documented in the runtime spec. `daemon.status()` reports `{ running, active, disabled, deferred, lastFireAt? }`; `daemon.metrics()` aggregates per-trigger success / error counters, catch-up applications, and library-mode warnings consumed by the Prometheus exposition.
- **Triggers REST routes.** `GET /v1/triggers` (scope `triggers:read`) and `GET /v1/triggers/:id` (scope `triggers:read`) serialize the persisted `TriggerState` rows verbatim so CLI + dashboard consumers can render them without re-mapping. `POST /v1/triggers/:id/fire` (scope `triggers:fire`) invokes the registered callback; `POST /v1/triggers/:id/disable` (scope `triggers:disable`) unregisters it; `POST /v1/triggers/prune` (scope `triggers:disable`) sweeps every disabled row.
- **Consolidator daemon (`@graphorin/server/consolidator`).** New `createConsolidatorDaemon({ consolidator })` lifecycle adapter starts the consolidator runtime in `beforeStart` and stops it in `beforeShutdown` with a hard 10-second timeout (force-aborts after the timeout so the daemon never hangs on a stuck phase). The adapter accepts a structurally-typed `ConsolidatorLike` so the server stays decoupled from the heavy `@graphorin/memory` package; consumers wire their own consolidator instance through `createServer({ consolidator })`.
- **Extended `/v1/health` (`@graphorin/server/health`).** The endpoint returns a typed rollup `'ok' | 'degraded' | 'failing'` plus a per-check breakdown carrying its detail fields **flat** on the same object as the `status` discriminator (no nested `detail` wrapper) — `storage: { status, walSizeBytes, warnThresholdBytes }`, `embedder: { status, modelLoaded, model? }`, `secrets: { status, activeStore }`, `encryption: { status, enabled, peerDepInstalled }`, `consolidator: { status, tier, running, paused, queueDepth, dlqSize, budgetRemaining: { tokens, costUsd } }`, `triggers: { status, running, active, disabled, deferred, lastFireAt? }`, `replayBuffer: { status, eventsBuffered, subscribers?, subscriptions? }`. HTTP 200 even when the rollup is `'degraded'` so liveness probes do not flap on minor degradations; `'failing'` short-circuits with 503.
- **`/v1/health/secrets` admin endpoint.** Mounted after the auth middleware so the scope check (`secrets:read`) has a verified token to inspect; returns `{ active, fallbackChain, downgradedFrom?, downgradeReason?, strictMode, headlessReasons }` per the secrets capability matrix. Backed by `getSecretsStoreStatus()` from `@graphorin/security/secrets` so the surface is bytes-equal between the server and the future `graphorin doctor --check-secrets` CLI command.
- **`/v1/metrics` Prometheus exposition (`@graphorin/server/metrics`).** New byte-stable `MetricRegistry` (counter / gauge / summary) renders Prometheus text exposition format v0.0.4 with quoted-escaped label values, NaN / `+Inf` / `-Inf` handling, and bounded-memory summary quantiles (p50 / p95). Strict label-name validation at sample time so PII never sneaks into a label key by accident. The canonical metric inventory ships pre-registered through `createServerMetricRegistry()`: `graphorin_agent_runs_total{status}`, `graphorin_agent_run_duration_seconds`, `graphorin_tool_calls_total{tool,status}`, `graphorin_provider_tokens_total{provider,model,direction}`, `graphorin_provider_cost_usd_total{provider,model}`, `graphorin_storage_wal_size_bytes`, `graphorin_idempotency_cache_hit_ratio`, `graphorin_triggers_fires_total{trigger_id,status}`, `graphorin_redaction_drops_total{reason,pattern}`, `graphorin_consolidator_queue_depth{phase}`, `graphorin_consolidator_dlq_size`, `graphorin_consolidator_budget_remaining_usd`, `graphorin_consolidator_budget_remaining_tokens`, `graphorin_oauth_tokens_freshness_seconds{server_id}`, `graphorin_replay_buffer_events`, `graphorin_inflight_runs`, `graphorin_server_uptime_seconds`, and a static `graphorin_build_info{version}` gauge. The registry is exposed on the `GraphorinServer.metrics` handle so consumer code (and tests) can drive sample updates programmatically. **Refresh-on-scrape**: every `/v1/metrics` request invokes a `refresh()` callback that samples live signals (WAL size, server uptime, in-flight runs, dispatcher size, consolidator status, trigger fire counters) into the registry just before rendering — Prometheus pull semantics without a background timer. No authentication required by default per the scrape-pattern convention; `server.metrics.requireAuth: true` opts into the `admin:metrics:read` scope for stricter deployments. Trigger-id label values are sanitized through `[^A-Za-z0-9_:.-]` → `_` to bound cardinality on user-supplied identifiers. Performance budget: 1000 metric samples render in well under the 50 ms ceiling documented in the runtime spec.
- **Scope-enforced replay endpoints (`@graphorin/server/replay`).** `POST /v1/runs/:runId/replay` and `POST /v1/sessions/:id/replay` now route through the new `createReplayRoutes(...)` family. `raw: false` (default) requires `traces:read:sanitized`; `raw: true` requires `traces:read:raw` (admin); missing scope returns 403 with a typed `scope-denied` body and a `hint` field pointing at the required scope. Every successful invocation appends a `replay:accessed` audit chain entry (mode + actor + target + event count); every refusal appends `replay:skipped` so operators can audit denied accesses. Audit failures never block the replay hot path. Consumer code supplies the `ReplayApi` (the framework knows how to enforce + audit; consumer code knows how to load trace events).
- **`POST /v1/audit/verify` chain integrity check.** New `audit:verify` scope; the endpoint delegates to the optional `AuditApi.verify(...)` adapter which wraps `verifyAuditChain()` from `@graphorin/security/audit`. Returns `{ verify: { ok, count? } }` on success or `{ verify: { ok: false, brokenAt } }` when the SHA-256 hash chain is broken. Returns 501 when the consumer opted out of chain verification.
- **`GET /v1/audit` filter parameters.** Extended to accept `?from=<iso|epoch>&to=<iso|epoch>&action=<id>&fromSeq=<n>&limit=<n>` per the runtime spec. Malformed timestamps return 400 with a typed `config-invalid` error. The `AuditApi.list({ fromTs, toTs, action, fromSeq, limit })` contract is forward-compatible — adapters that ignore the new keys keep working without changes. `POST /v1/audit/export` accepts `format: 'jsonl' | 'csv'` (body or query) so consumers can pick the export shape.
- **`server.metrics` + `server.health` config blocks.** `metrics.{ enabled, path, requireAuth }` (defaults `true` / `'/metrics'` / `false`) — the metrics endpoint mounts under `${basePath}${metrics.path}` (default `/v1/metrics`) and skips auth when `requireAuth: false` matches the documented Prometheus scrape pattern. `health.{ walWarnThresholdBytes }` (default 50 MB) governs the storage check warn threshold.
- **`GraphorinServer` handle extensions.** New `triggers`, `consolidator`, `metrics` accessors on the handle returned by `createServer(...)`. `triggers` and `consolidator` are populated only when the operator wired the corresponding option (`triggers: { scheduler | daemon }` / `consolidator: ConsolidatorLike`); `metrics` is always present (the canonical registry is constructed by default).
- **`@graphorin/server/health`, `@graphorin/server/metrics`, `@graphorin/server/triggers`, `@graphorin/server/consolidator`, `@graphorin/server/replay` sub-paths.** Consumers can import the primitives without pulling the full server.
