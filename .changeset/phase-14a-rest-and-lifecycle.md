Phase 14a â€” initial release of `@graphorin/server` (the standalone server runtime) and `@graphorin/cli` (the operator CLI). Together they ship the REST surface, lifecycle plumbing, auth + scope + idempotency + audit middleware stack, and the minimal `graphorin start / init / migrate` binary that lets a Graphorin deployment run as a long-living daemon.

`@graphorin/server` ships:

- **`createServer({...})` programmatic entry point.** Builds a Hono app, registers the request-state, CORS, CSRF, rate-limit, auth, scope, idempotency, and audit middleware in the documented order, mounts every REST route, exposes the `AgentRegistry` + `WorkflowRegistry` + `RunStateTracker` so consumer code can plug agents and workflows in without taking a peer dependency on `@graphorin/agent` / `@graphorin/workflow`.
- **`defineConfig(...)` + `parseServerConfig(...)`.** Strongly-typed `graphorin.config.ts` loader backed by a Zod schema; every field has a documented production-ready default; invalid inputs surface a typed `ConfigInvalidError` carrying the exact failing path.
- **Lifecycle plumbing.** `beforeStart` â†’ pre-bind validation â†’ storage migrations â†’ bind â†’ `onReady`; SIGTERM/SIGINT â†’ `beforeShutdown` â†’ drain â†’ exit. Pre-bind fails fast on missing pepper, unresolvable `SecretRef`, missing encryption peer (when required), or migration failures â€” each emits a typed error with a `hint` field pointing at `graphorin doctor --check-secrets` (Phase 15) or the equivalent remediation step. `applyProcessHardening` (DEC-135) runs in `beforeStart` so the daemon refuses to run as root and inherits a `0o077` umask before binding the listener.
- **REST endpoints.** Agents (`run`, `stream`, `state`, `abort`, `resume`), workflows (`execute`, `resume`, `state`, `checkpoints`, `fork`), sessions (`create`, `list`, `get`, `delete`, `messages`, `handoffs`, `export`, `replay`), memory (`search`, `facts`, `blocks`), skills (`list`, `install`, `get`), MCP (`servers`), tokens (`create`, `list`, `revoke`), audit (`list`, `export`), and `/health`. Every side-effecting endpoint honours an `Idempotency-Key` per IETF draft-07: cached responses replay verbatim with `Idempotency-Replayed: true`; mismatched payloads return `409`; expired records re-execute.
- **Auth + scope middleware.** Bearer-token verification routed through the `TokenVerifier` from `@graphorin/security` (HMAC-SHA256 + pepper, brute-force lockout, warm-path LRU cache). The verified token is mirrored onto the documented `c.var.token: { id, label, scopes, env, expiresAt }` slot so route handlers and downstream consumers can read it without unwrapping the auth discriminator. Scope grammar `<resource>:<action>[:<id>]` enforced per-route through a small `createScopeMiddleware(...)` factory; `admin:*` is the single super-scope.
- **CORS / CSRF / rate limit.** Deny-by-default CORS (no echoing of arbitrary origins; explicit allowlist semantics); double-submit CSRF for browser flows (bearer-authenticated requests are exempt); per-IP sliding-window rate limit with `X-RateLimit-*` headers.
- **Audit middleware.** Forwards every authenticated request through the tamper-evident chain (`appendAudit` from `@graphorin/security/audit`); records actor, action `http:request` (matching the Phase 14a spec), target, decision (`success` / `denied` / `error` / `not-found` derived from the response status), request-id, idempotency-key, idempotency-replayed flag, and duration. Append failures never break the request hot path; they are surfaced through a configurable `onError` sink.
- **Replay scope ladder.** `POST /v1/sessions/:id/replay` enforces `traces:read:sanitized` for the default sanitized-replay path and `traces:read:raw` when the caller opts into raw replay (DEC-138 / ADR-032). The 14a handler returns the 202 + cursor envelope so the actual stream wires up cleanly when Phase 14b lands.
- **Graceful shutdown.** SIGTERM aborts every reserved-but-not-yet-running run immediately (so streaming-endpoint reservations never block the drain), then waits up to `server.shutdown.drainTimeoutMs` (default 30 s) for in-flight work to complete. After the timeout the tracker aborts everything that didn't drain in time so state is preserved per DEC-143.
- **Production-hardened defaults.** Refuses to bind the listener until pre-bind validation passes; opt-in mandatory-encrypted `audit.db` (DEC-124); per-route Zod-validated request bodies; typed error surface with stable `kind` discriminators + `hint` fields covering every documented failure mode.

`@graphorin/cli` ships:

- The single `graphorin` binary (built on top of `commander@^10.0.0`; ESM-only; `#!/usr/bin/env node` shebang prepended at build time). `graphorin --help` lists the three commands grouped by lifecycle phase (Bootstrap / Maintenance / Runtime) with a top-of-help reminder that `GRAPHORIN_OFFLINE=1` is honoured by every subcommand.
- **`graphorin init`** â€” interactive bootstrap. Generates a fresh `graphorin.config.ts` with documented defaults; prompts for the cloud-upload sensitivity tier (`public-only` / `public-and-internal` / `all-with-warnings`) and the storage-encryption opt-in; mints a server pepper (printed to stderr exactly once for the operator to persist via their preferred secret store) and a one-shot bootstrap admin token. `--non-interactive` accepts every option through CLI flags or environment variables (`GRAPHORIN_CLOUD_CONSENT`, `GRAPHORIN_STORAGE_ENCRYPTED`).
- **`graphorin migrate`** â€” applies every pending storage migration against the configured SQLite store. Idempotent: re-running the command after every migration has been applied is a no-op + a successful exit. Reserves the `--target <version>` flag for the Phase 15 partial-migration runner.
- **`graphorin start`** â€” boots the standalone server. Honours `--config <path>`, `--host <host>`, `--port <port>`, plus the DEC-136 `--secrets-source <kind>` (auto / keyring / encrypted-file / env) and `--strict-secrets` flags that override the corresponding `secrets` block of the loaded config. Installs SIGTERM/SIGINT handlers that delegate to `server.stop()` for the graceful drain. Never prints raw secret values; lifecycle errors surface with the underlying `kind` discriminator + `hint` field.
- **`GRAPHORIN_OFFLINE=1` enforcement.** Exposes `isOfflineMode()` + `assertNoNetworkInOfflineMode()` + `OfflineModeViolationError` so Phase 15 commands that touch the network (`graphorin pricing refresh`, etc.) fail fast in offline mode. Phase 14a's three commands print a short notice when the flag is set so the operator confirms the local-only execution path.
